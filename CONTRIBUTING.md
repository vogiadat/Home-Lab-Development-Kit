# Contributing

This project is maintained as a documentation-first deployment kit.

## Guidelines

- Keep documentation practical and deployment-focused.
- Prefer runnable commands and verification steps over long theory sections.
- Keep scripts modular, readable, and idempotent where practical.
- Do not commit generated release files unless they are part of a tagged release.
- Store editable diagrams in `diagrams/` and rendered images in `assets/images/`.

## Commit Style

Use concise commit messages:

```text
docs: add Windows host optimization chapter
scripts: add Hyper-V NAT setup module
ci: add PDF build workflow
```
